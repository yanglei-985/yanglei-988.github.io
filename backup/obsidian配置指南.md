# Obsidian + Claude Code 教程整理（Step by Step）


> - **Obsidian CLI**
> - **Claude Code**
> - **Claudian**
> - **Skills**

## 这套组合到底是什么？

- **Obsidian**：你的知识库与笔记空间
- **Claude Code**：负责理解指令、分析内容、执行操作的 AI 助手
- **Claudian**：面向 Obsidian 使用场景定制的 Claude/Claude Code 助手
- **Obsidian CLI**：让 AI 更顺畅地和 Obsidian 配合
- **Skills**：把常见工作流封装成可复用能力

一句话理解：

> Obsidian 负责存知识，Claude Code 负责理解和执行，Skills 负责把高频任务变成“可复用动作”。

---

## Step 0：开始前先确认 3 个前提

在正式开始前，先确认你已经具备下面这些条件：

1. 已安装并更新 **Obsidian**
2. 已经能正常使用 **Claude Code**
3. 已配置好可用的 **API / 模型**



## Step 1：升级 Obsidian，并开启 Obsidian CLI

### 操作步骤

1. 打开 Obsidian
2. 升级到最新版本
3. 进入 **设置 → 关于**
4. 找到命令行界面 / CLI 相关功能
5. 按提示完成注册
6. 重启 Obsidian

### 这一步的意义

- 让 Obsidian 可以被命令行能力调用
- 让 Claude Code 不只是“读文件”，还能更深入地和 Obsidian 配合

---

## Step 2：安装 Claudian 插件

### 操作步骤

1. 打开 **Claudian 插件**的 GitHub 仓库https://github.com/YishenTu/claudian
2. 点击右侧 **Releases**
3. 下载最新版本中的 3 个文件
4. 打开你的 Vault 中的 `.obsidian/plugins/`
5. 新建对应的插件文件夹
6. 把刚才下载的 3 个文件放进去
7. 重启 Obsidian
8. 打开 **设置 → 第三方插件**
9. 启用 Claudian
10. 看到侧边栏图标后，打开对话界面

> [!tip]
> 如果你找不到 `.obsidian` 或 `plugins` 文件夹，先打开系统里的“显示隐藏文件夹”选项。

### 这一步完成后，你应该得到什么？

- Obsidian 侧边栏出现 Claudian 图标
- 你可以直接在 Obsidian 内和 Claude Code 风格的助手对话

### 注意

想在这里顺利使用 Claudian，前提仍然是：

- Claude Code 本体已可用
- API / 模型配置已经完成

---

## Step 3：安装 Obsidian Skills

### 操作步骤

1. 打开 **Obsidian Skills** 的 GitHub 仓库
2. 下载压缩包并解压
3. 找到其中推荐的 5 个 Skill
4. 打开 Claudian / Claude Code 对应的 `skills` 目录
5. 把这 5 个 Skill 文件夹复制进去
6. 重启或重新加载工具

### Skills 的作用

- 把高频任务固化为现成能力
- 以后可以直接调用，不用每次重新写复杂提示词
- 适合做结构化输出、画板、数据库、整理资料等任务

---

## Step 4：先做 3 个最小测试，确认整套流程已经跑通

建议按下面顺序测试。

### 测试 1：创建一篇标准化笔记

你可以这样说：

> 帮我创建一篇名为《纳瓦尔宝典》的读书笔记，并按照我现有的读书笔记风格填写内容。

理想结果：

- 笔记被创建到了正确位置
- 使用了正确模板或接近你现有风格
- AI 会参考你已有的相似笔记，而不只是机械套模板

### 测试 2：整理一篇收藏资料

你可以这样说：

> 帮我整理这篇 Python 收藏文档：去重、归类，并给我一个清晰目录。

理想结果：

- 资料被重新分组
- 重复内容被处理
- 文档结构变得更容易阅读

### 测试 3：调用 Skill

例如：

- 用画板 Skill 把这篇文章整理成思维导图
- 帮我生成一个“2025 年读书笔记汇总数据库”

理想结果：

- AI 会主动判断什么时候该调用 Skill
- 复杂任务能明显减少手工操作

## Step 5：理解两个关键工作模式

### 模式 1：自动执行 / YOLO 模式

特点：

- AI 会连续自动执行操作
- 可能直接修改、整理甚至删除文件

适合：

- 你已经非常确定目标是什么
- 任务比较标准化

风险：

- 改动会来得很快
- 如果你没盯着看，可能已经改完了

### 模式 2：Plan 模式（先方案，后执行）

特点：

- AI 先列出执行方案和步骤
- 你可以先讨论、修改，再让它正式执行

适合：

- 工作流搭建
- 批量整理
- 大范围改写
- 结构调整类任务

### 建议用法

- **简单任务**：普通模式即可
- **复杂任务**：优先用 Plan 模式

## Step 6：学会几个高频实用技巧

### 1. 调整思考等级

- 简单任务：低等级或不开思考
- 复杂任务：提高思考等级
- 代价：等级越高，通常越耗 token

### 2. 上传文件一起分析

- 对话框下方的文件夹图标可以上传文件
- 适合把“不在 Vault 里的资料”一起交给 AI 分析

### 3. 用斜杠命令调用 Skill 或系统能力

- 可以快速调用指定 Skill
- 适合高频固定动作

### 4. 指定文件 / 框选提问

- 可以针对某个文件单独处理
- 也可以只框选一段文字，做定向分析或改写

## Step 8：安全建议（非常重要）

1. **大改之前先备份**
2. **最好给 Vault 建 Git 仓库，方便回滚**
3. **新工作流先在测试文件夹试**
4. **不确定时关闭自动执行，改用逐步确认**
5. **如果改崩了，优先回滚版本，不要让 AI 硬改回去**

> [!warning]
> 真正浪费时间和 token 的，往往不是第一次修改，而是“改坏后再补救”。  
> 所以：先备份，再放手让 AI 动手。

---

## 最小上手路径（第一次用就照这个来）

如果你只想先把整套流程跑通，可以按下面做：

1. 更新 Obsidian，并开启 CLI
2. 安装 Claudian
3. 确认 Claude Code / API 可用
4. 安装 Skills（可选，但很推荐）
5. 先让 AI 创建一篇标准化笔记
6. 再让它整理一篇已有收藏文档
7. 最后尝试一个 Skill（思维导图或数据库）

---

## 一句话总结

这套系统的重点，不是“把 AI 放进 Obsidian 聊天”，而是让 AI 真正参与到你的知识整理、写作和内容生产流程里。
